Git과 GitHub는 왜 중요할까요?

효율적 파일 관리

코드를 포함한 파일의 변경 사항을 체계적으로 관리하고 싶을 때

팀 협업

여러 사람이 동시에 같은 프로젝트에서 작업하고 싶을 때

안전한 백업

작업 내용을 안전하게 백업하고 언제든 이전 상태로 되돌아가고 싶을 때

글로벌 협업

전 세계 개발자들과 협업하고 코드를 공유하고 싶을 때

'팀 작업 + 타임머신 + 온라인 작업실' = Git + GitHub

용어 1. Git

버전 관리의 마법사

핵심 설명

  • 오픈 소스 버전 제어 시스템(VCS)
  • 파일의 변경 이력을 기록하고 관리
  • 주로 로컬 컴퓨터에서 작업
💾

비유

'게임 중간 저장' 기능

또는 글쓰기 중 '수정 이력 남기기'

용어 2. GitHub

온라인 협업 플랫폼

핵심 설명

  • Git을 기반으로 하는 웹 기반 호스팅 서비스
  • 코드를 저장, 공유, 협업하는 온라인 공간
  • 팀 프로젝트 관리, 코드 리뷰 기능 제공
☁️

비유

'팀 프로젝트를 위한

클라우드 드라이브 +

소셜 네트워킹 기능'

용어 3. Repository

프로젝트의 모든 것

핵심 설명

  • 프로젝트의 모든 파일과 변경 이력(커밋)이 저장되는 공간
  • 로컬 리포지토리: 내 컴퓨터에 있는 복사본
  • 원격 리포지토리: GitHub 서버에 있는 원본 (협업의 중심)
📁

비유

'프로젝트 폴더 +

변경 이력 앨범'

용어 4. Commit

변경 사항 저장하기

핵심 설명

  • 파일의 변경 사항을 기록하는 작업 단위
  • 각 커밋은 특정 시점의 프로젝트 스냅샷
  • 커밋 메시지: 왜 변경했는지 설명 (중요!)
💾

비유

'문서 작업 중

'저장' 버튼 누르기

(메모와 함께)

용어 5. Push

로컬 변경사항 올리기

핵심 설명

  • 로컬 리포지토리의 커밋들을 원격 리포지토리로 전송
  • 내 작업을 다른 사람들과 공유하는 과정
☁️

비유

'내 컴퓨터에서 작업한 내용을

클라우드 서버에 업로드하기'

용어 6. Branch

독립적인 작업 공간 만들기

핵심 설명

  • 메인 코드라인에서 분리되어 독립적으로 개발 진행
  • 메인 코드에 영향을 주지 않고 새 기능 개발, 버그 수정 등에 활용
  • 작업 완료 후 다시 메인 코드와 합침
🌿

비유

'메인 스토리와 분리된

스핀오프 작업'

또는 '여러 갈래 길 중
하나를 선택하여 나아가기'

용어 7. Pull Request

변경사항 병합 요청 및 검토

핵심 설명

  • 내 분기의 변경 사항을 다른 분기(예: main)에 병합해 달라고 요청
  • 코드 리뷰, 토론, 추가 수정 등 협업 과정의 핵심
🤝

비유

'팀원에게 내 작업 결과물을

보여주며 최종 버전에 반영해달라고

요청하고 피드백 받기'

용어 8. Merge

두 갈래 길을 하나로 합치기

핵심 설명

  • 서로 다른 분기의 변경 사항을 하나로 합치는 작업
  • 주로 Pull Request가 승인된 후 발생
🔗

비유

'스핀오프 스토리를

다시 메인 스토리에

합치는 것'

용어 9. Fork

다른 프로젝트 가져와서 내 것으로 만들기

핵심 설명

  • 다른 사용자의 원격 리포지토리 전체를 내 GitHub 계정으로 복사
  • 원작에 기여하거나 나만의 프로젝트로 발전시킬 때 사용
🍴

비유

'마음에 드는 오픈소스 레시피를

보고 내 레시피 노트에 옮겨 적어

나만의 재료를 추가해보기'

용어 10. Clone

원격 리포지토리를 내 컴퓨터로 가져오기

핵심 설명

  • 원격 리포지토리의 코드를 로컬 컴퓨터로 다운로드
  • 로컬 리포지토리를 생성하여 작업 준비
📥

비유

'클라우드 드라이브에 있는

문서를 내 컴퓨터로

다운로드하기'

추가 용어 1: 이슈와 지스트

Issue (이슈)

  • 프로젝트 관련 작업, 버그, 질문 등을 추적/관리하는 기능
  • 할 일 목록이나 토론 공간으로 활용
비유: '프로젝트 할 일 목록/게시판'

Gist (지스트)

  • 짧은 코드 스니펫이나 메모를 공유하는 기능
  • 간단한 정보 공유에 유용
비유: '간단한 코드 메모장 공유'

추가 용어 2: Git이 무시할 파일들

Ignoring files, .gitignore

핵심 설명

  • Git 버전 관리에서 제외할 파일/폴더 지정
  • .gitignore 파일에 목록 작성
  • 로그 파일, 빌드 결과물, 개인 설정 파일 등 불필요한 파일 관리 방지
🚫

비유

'버전 관리 청소 목록'

또는 '백업하지 않을 파일 목록'

사용자 및 관리 1: 누가 GitHub를 사용하나요?

Account (계정)

  • GitHub를 사용하는 개인 또는 조직
비유: '개인 회원 가입'

Organization (조직)

  • 여러 개인 계정이 모여 팀 프로젝트를 관리하는 계정 유형
  • 협업 효율성 증대 및 권한 관리 용이
비유: '회사 또는 팀 계정'

사용자 및 관리 2: 어떤 기능과 권한이 있나요?

Plan (플랜)

  • 기능 및 서비스 수준에 따른 요금제 (Free, Pro, Team 등)
비유: '서비스 이용 등급'

Access Permission (액세스 권한)

  • 리포지토리 등 리소스에 대한 사용자의 작업 권한 (읽기, 쓰기, 관리 등) 설정
비유: '파일/폴더 접근 권한 설정'

요약 및 다음 단계

핵심 개념 정리

  • Git: 로컬 버전 관리 도구
  • GitHub: 온라인 협업 플랫폼
  • Repository: 프로젝트 저장소
  • Commit/Push: 변경사항 기록/전송
  • Branch: 독립적 작업 공간
  • Pull Request/Merge: 협업과 병합
